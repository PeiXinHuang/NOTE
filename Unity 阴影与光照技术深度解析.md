# 概念解析
## 一、 核心技术概念对比
在计算机图形学和 Unity 中，处理物体遮挡光线（阴影）主要有三类核心方案：

| 技术名称 | 核心原理 | 适用对象 | 优点 | 缺点 |
|---|---|---|---|---|
| 阴影映射 (Shadow Mapping) | 从光源视角生成深度图，实时对比像素深度 | 动态物体 (角色、车辆) | 实时更新，位置准确 | 性能开销大，易产生锯齿和失真 |
| 光照贴图 (Lightmapping) | 将光影信息预先计算并“画”在物体纹理上 | 静态物体 (建筑、地形) | 性能极高，支持复杂的间接光反弹 | 无法实时移动，烘焙时间长 |
| 光照探针 (Light Probes) | 在空间中布置采样点，记录环境光信息 | 动态物体 (受静态环境影响) | 让动态物体融入烘焙环境 | 精度低，无法处理精细阴影切割 |

------------------------------
## 二、 阴影映射 (Shadow Mapping) 详解
## 1. 工作流程 (Two-Pass)

* Pass 1 (Depth Pass)：相机移至光源位置，记录场景中物体到光源的距离（深度值），生成 深度贴图 (Depth Map)。
* Pass 2 (Lighting Pass)：正常渲染时，将每个像素到光源的实际距离与深度贴图中的值对比。如果“实际距离 > 贴图值”，则该点处于阴影中。

## 2. Unity 中的实现

* 级联阴影 (CSM)：Unity 将视锥体分段，近处用高分辨率贴图，远处用低分辨率，平衡性能与质量。
* 阴影偏置 (Bias)：通过微调深度对比值，解决“阴影失真 (Shadow Acne)”问题。

------------------------------
## 三、 光照贴图 (Lightmapping) 详解
## 1. 为什么静态物体用它？
静态物体位置固定，重复每一帧计算深度对比是浪费性能的。通过烘焙 (Baking)，可以将光线反弹（全局光照 GI）的效果永久保存在一张特殊的贴图里。
## 2. 混合模式 (Mixed Lighting)
这是 Unity 最常用的模式：

* 静态物体显示烘焙好的精致光影。
* 动态物体的实时阴影可以叠加在静态物体（地面）上，实现“动静结合”。

------------------------------
## 四、 光照探针 (Light Probes) 详解## 1. 解决的核心问题
当动态物体（没有烘焙数据）走进静态物体的阴影区（没有实时光）时，物体会显得突兀地过亮。光照探针负责告诉动态物体：“你当前所在的空间光线很暗”。
## 2. 局限性与补救

* 局限：基于点采样，无法实现“半身亮半身暗”的精确切割。
* 补救：
* 增加密度：在门窗等光影剧变处密集摆放。
   * LPPV：为大型物体提供网格化的采样场，提升精度。

------------------------------
## 五、 Unity 实践指南：如何布置与生效
## 1. 物体与光源设置

* 静态物体：在 Inspector 面板右上角勾选 Static。
* 动态物体：确保 Mesh Renderer -> Receive Shadows 开启，且 Light Probes 设置为 Blend Probes。
* 光源模式：
* Realtime：全实时。
   * Baked：全烘焙。
   * Mixed：自动分流动静物体。

## 2. 光照探针的布置步骤

   1. 创建组：Create -> Light -> Light Probe Group。
   2. 编辑位置：点击 Edit Light Probes，在光影变化处（如窗户边缘、树荫下）手动复制并放置小球。
   3. 避坑指南：不要将探针放进墙体内部或地板下方。
   4. 生成数据：在 Lighting 窗口点击 Generate Lighting 开始烘焙。

## 3. 运行逻辑链条

   1. 判断身份：Unity 扫描场景，区分 Static 和动态物体。
   2. 静态渲染：直接读取 Lightmap。
   3. 动态投射：实时计算动态物体的 Shadow Map 并叠加到场景中。
   4. 动态着色：根据物体位置对周围的 Light Probes 进行插值，改变物体底色。


# Unity 阴影性能优化指南
在 Unity 中，阴影通常是渲染开销最大的部分之一。优化核心在于：减少参与计算的像素、降低计算频率、以及利用预计算替代实时计算。
## 一、 全局参数优化 (Global Settings)
这些设置通常在 Project Settings > Quality 或 URP/HDRP Asset 中调整。

| 参数 | 优化建议 | 影响 |
|---|---|---|
| Shadow Distance | 最优先项。按需缩减（如从 150m 降至 40m）。 | 超出距离的物体不再渲染实时阴影，极大降低 Draw Call。 |
| Shadow Cascades | 移动端建议 0-2 级，PC 端 4 级。 | 级联越多，阴影在远近过渡越平滑，但每增加一级都会翻倍渲染开销。 |
| Shadow Resolution | 减半设置（如 2048 降至 1024）。 | 降低显存占用和像素填充率压力。 |
| Soft Shadows | 性能敏感场景改用 Hard Shadows。 | 软阴影需要多次采样，计算量比硬阴影大。 |

------------------------------
## 二、 光源与物体层面的优化
## 1. 减少实时计算量

* 光源剔除 (Culling Mask)：确保光源只照射必要的层级（Layer），防止不重要的物体（如小草、碎石）触发阴影渲染。
* 减少实时光数量：场景中尽量只保留一盏 Mixed 或 Realtime 的主方向光（太阳），其余补光使用 Baked 模式。

## 2. 阴影代理 (Shadow Proxy) —— 高级技巧

* 原理：用低模投射阴影，高模渲染显示。
* 操作：
1. 准备一个极低面数的模型作为“替身”，放在同一位置。
   2. 将高模的 Cast Shadows 设为 Off。
   3. 将低模的 Cast Shadows 设为 On，但取消勾选其渲染显示的材质，或设为隐藏层。
* 效果：阴影轮廓几乎不变，但显卡计算深度图的压力锐减。

## 3. Mesh Renderer 微调

* Cast Shadows：对视野外或不明显的物体直接设为 Off。
* Receive Shadows：对不需要显示别人影子的物体（如特效、天空盒）设为 Off。

------------------------------
## 三、 烘焙与混合技术 (Baking)

* 使用 Shadowmask 模式：这是性能与画质的最佳平衡。静态影子的遮挡关系被烘焙在纹理中，而动态物体依然能产生实时投影。
* 光照探针 (Light Probes)：用探针替代动态物体的复杂光照计算，让角色在静态阴影区自然变暗，而无需昂贵的全局实时光。

------------------------------
## 四、 阴影瑕疵修复 (无损性能)
通过调整 Light 组件上的 Bias 参数，在不增加开销的前提下提升视觉质量：

* Shadow Bias：解决“阴影失真/暗斑 (Shadow Acne)”，若地面有条纹则调大。
* Normal Bias：解决“漏光”，防止阴影穿透墙角。
* Near Plane：调大光源的近裁剪面，可以过滤掉光源背后或过近物体的无效阴影。

------------------------------
## 五、 优化工作流检查清单

   1. 确定目标平台：手机端必须严格控制 Shadow Distance（建议 < 50）。
   2. 静态物体勾选 Static：确保它们不参与每帧的 Shadow Map 刷新。
   3. 检查 Draw Call：查看 Frame Debugger，确认是否有非必要的物体在投射阴影。
   4. 评估分辨率：阴影锯齿感严重吗？如果严重，优先调偏置（Bias）而非无脑增加分辨率。



* 性能优先：尽量烘焙静态物体。
* 画质优先：在关键区域增加光照探针密度，并开启 URP/HDRP 的高级阴影特性（如 PCSS 软阴影）。
